# **DevLens: Frontend Architecture & UI Design Guide**

This document outlines the detailed strategy for building the DevLens frontend. The web application serves as the "Cockpit" for developers trying to understand and contribute to an open-source project. 

**Tech Stack:** 
- **Core:** React.js (Next.js or Vite for tooling)
- **Styling:** Tailwind CSS (for rapid, modern styling)
- **Visualization:** `react-force-graph` (for the 3D/2D dependency graph)
- **Animations:** `framer-motion` (for smooth transitions and a premium feel)
- **State Management:** `Zustand` (for handling complex graph nodes, agentic chat context, and user settings).
- **Code Highlighting:** `prismjs` or `highlight.js`

---

## **1. The Layout Architecture (The "DevLens" Cockpit)**

The UI should feel like a modern IDE combined with an intelligent assistant. 

### **A. Primary Layout Structure**
- **Left Sidebar (The Navigator):** 
  - Repository Input & Ingestion Status.
  - "Good First Issue" Matcher feed.
  - Environment Setup Scripts viewer.
- **Center Canvas (The Map):**
  - **View 1 (Default):** The Interactive Dependency Graph (`react-force-graph`).
  - **View 2 (Code Focus):** A split view showing the Graph on top and the Code Editor/Viewer on the bottom.
- **Right Sidebar (The DevLens Architect):**
  - The AI Agent Chat interface (Phase 7).
  - Jargon Buster / Explainer tools.
  - Architectural Intent displays.

## **2. Feature-by-Feature UI Implementation**

### **Feature A: The Gatekeeper & Repository Ingestion (The Entry Point)**
**Backend API:** `GET /api/v1/gatekeeper`, `POST /api/v1/repository/ingest`
- **UI Element:** A clean, centered hero screen (when no repo is loaded) with a prominent input box: *"Paste a GitHub URL to begin."*
- **Gatekeeper Flow (Phase 8):** Before downloading, the UI hits the `/gatekeeper` endpoint.
  - If the repo is dead (>1 yr old), highly competitive (>50 PRs), or overly complex (>500 deps), display a styled Warning overlay or Toast letting the user know the risks before proceeding.
- **Animation:** Once "Ingest" is confirmed, show a `framer-motion` loading state (e.g., scanning code blocks, parsing AST).

### **Feature B: User Context Engine (The Onboarding Modal)**
**Frontend State:** Zustand (`user_profile`)
- **UI Element:** A sleek "One-Minute Calibration" modal that pops up on first visit.
- **Fields:**
  - **Level:** [Student / Junior / Senior]
  - **Language:** [English / Hindi / Hinglish]
  - **Goal:** [Learning / Contributing]
- **Mechanism (Phase 8):** This data is stored in Zustand and explicitly passed as a JSON header in all requests to the backend, enabling the AI to dynamically shift its tone without requiring a database.

### **Feature C: Anti-Gravity Handover & Setup Guidance**
**Backend API:** `GET /api/v1/setup/{owner}/{repo}`
- **UI Details:** Once ingested, a modal or a dedicated tab in the Left Sidebar reveals the **"1-Click Setup"**.
- **The Upgrade (Phase 8):** It generates precise, copy-pasteable terminal blocks (e.g., `git clone`, `poetry install`, `git checkout -b`).
- **Safety Badge:** A visible alert warning the user to check their local environment prerequisites (e.g., "Requires Python 3.10+") before pasting the commands.

### **Feature B: The Dependency Graph (The Map)**
**Backend API:** `GET /api/v1/repository/graph/{owner}/{repo}`
- **Tool:** `react-force-graph-2d` or `react-force-graph-3d`.
- **UI Details:** 
  - Render Nodes (files) as specific colors based on their language or type. 
  - Node size should be proportional to file complexity (size in bytes).
  - Create a "Hover Overlay" using `framer-motion`: When a user hovers over a node, display file name, size, and incoming/outgoing imports.
  - **Click Action:** Clicking a node opens that file in the **Code Focus View** and pre-fetches its Architectural Intent.

### **Feature E: Architectural Intent (Institutional Memory)**
**Backend API:** `POST /api/v1/intent`, `GET /api/v1/history/{owner}/{repo}`
- **UI Details:** 
  - When a file node is clicked, the Right Sidebar displays a **"File History"** tab.
  - Show a stylized timeline of the most recent PRs that touched this file.
  - Show the 3-4 paragraph "Architectural Intent" summary generated by OpenRouter (Nemotron-3) analyzing *why* this file exists.

### **Feature F: The Indic Bridge (Multilingual Explainer & Jargon Buster)**
**Backend API:** `POST /api/v1/explain`
- **UI Details:** 
  - In the Code Focus View, a user can highlight lines of code.
  - A tooltip pops up: **"Explain this"**.
  - Clicking this opens a modal or the Right Sidebar containing the Jargon Buster.
  - **The Tone/Language Shift:** The backend utilizes the `user_profile` headers to automatically set the language and complexity level. A sleek dropdown still exists to manually override the language (Hindi, Tamil, Hinglish, etc.).

### **Feature G: The "Good First Issue" Matcher**
**Backend API:** `GET /api/v1/issues/recommend/{owner}/{repo}`
- **UI Details:**
  - Placed in the Left Sidebar under an "Available Missions" tab.
  - Displays a clean feed of `good first issue` cards fetched from GraphQL.
  - **Smart Badges:** Use Tailwind colors to indicate status:
    - 🟢 "Available"
    - ⚠️ "In Progress" (if the backend detected linked active PRs).
  - Clicking an issue card pushes it to the **DevLens Architect Agent**.

### **Feature G: The "DevLens Architect" (Agentic Contribution Workflow)**
**Backend Core:** Phase 7 integration (Agentic Mode, Context Passing)
- **UI Element:** The persistent Right Sidebar Chatbot UI. 
- **Workflow:**
  1. **Issue Selected:** The user clicks an issue from the Left Sidebar. The Chatbot wakes up and declares: *"I've analyzed Issue #42. Based on the Graph, the blast radius is auth.py and user_profile.ts. Here is your step-by-step contribution plan."*
  2. **Tactical Commands:** Render terminal commands provided by the AI inside beautifully styled code blocks with a **"Copy Command"** button.
  3. **Terminal Feedback Loop:** Add a subtle text input beneath the chat: *"Paste terminal output (errors/logs) here."* If the user pastes an error, it sends `type="terminal_output"` to the backend so the agent seamlessly debugs the error without forgetting the main mission context.

## **3. Design Aesthetics & UX Principles**

- **Color Palette:** Dark mode by default for developers.
  - Background: Deep slate (`bg-slate-900` or `#0F172A`).
  - Accents: Neon Cyan (`#06b6d4`) or Indigo (`#6366f1`) for buttons, active graph nodes, and AI highlighting.
- **Micro-Interactions:** 
  - Use `framer-motion` for staggering list items (like the PR history or Setup scripts) so they glide in gracefully.
  - Graph zoom/pan should feel buttery smooth.
- **Error Handling:** If the backend hits rate limits or large file warnings, use non-intrusive Toast notifications (e.g., `react-hot-toast`) over hard modal dialogs so the user's flow isn't interrupted.
