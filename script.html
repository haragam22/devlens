03 What DevLens Is 45 seconds

The pitch

To solve this, we built DevLens.

DevLens is an AI-powered onboarding navigator. It doesn't write your code for you — it builds your architectural intuition.

You give it a GitHub URL. In under 60 seconds, our backend clones the repo, parses the complete Abstract Syntax Tree, generates vector embeddings for every function, and renders a fully interactive, deterministic 3D map of the entire architecture.

It acts as a digital Senior Architect. It tells you exactly where to look, why the code was written that way, and what you are about to break.

04 The Engineering Features 2 minutes
🖥️ If you can show 44.202.1.36 on screen — do it here. Point at things as you talk. Frame these as workflow upgrades.
Feature 1 — Deterministic Dependency Mapping

The first thing you see is the Molecular Map.

Unlike standard RAG apps that just dump text into an LLM and pray it doesn't hallucinate, our map is mathematically verified. We use Tree-sitter to parse the AST and build a concrete node-and-edge graph of imports and classes. You visually see how the entire backend couples with the frontend.

Feature 2 — Semantic Issue Mapping

Say you grab a GitHub issue: "Search crashes on special characters."

Paste that into DevLens. We run a hybrid vector and BM25 search against the chunked codebase. It skips the manual `grep` and immediately highlights the exact file and function you need to fix.

Feature 3 — Blast Radius (The Killer Feature)

This is the feature every developer wishes they had. Before you touch that auth file, DevLens runs a graph query and highlights your Blast Radius.

It explicitly warns you: "If you modify this token validator, you will break these three downstream services." It protects junior developers from taking down `main`.

Feature 4 — Institutional Memory (RAG on PRs)

Why was a weird block of code written? DevLens queries the GitHub GraphQL API, pulls the last 50 merged Pull Requests and their code-diffs, and maps them to the AST.

It tells you: "This messy code was added in PR #402 to fix a race condition. Do not refactor it out." We give day-one developers the institutional memory of a senior engineer.

Feature 5 — The Indic Bridge (Built for Bharat)

Finally, we have to talk about accessibility. For a student in a Tier-2 or 3 college, reading a massive `CONTRIBUTING.md` full of jargon like "idempotent eventually consistent APIs" is demoralising.

Toggle "Junior Mode" in DevLens. It passes the dense documentation through OpenRouter, simplifies the tone, and translates it into regional languages.

That sentence becomes: "Yeh service ek traffic cop jaisi hai. Agar same request baar baar aaye, toh woh duplicate kaam nahi karegi." We bridge the gap between complex tech and regional talent.

05 Architecture & Guardrails 45s in pitch · full Q&A below
⚙️ Hit the stack hard and fast. Prove this isn't just a basic OpenAI wrapper.
SAY THIS IN THE MAIN PITCH

The stack is built for scale and resilience.

Frontend is React + Tailwind + React Force Graph for the CLI visualizer. Backend is an asynchronous FastAPI layer deployed on AWS EC2.

For the AI layer, we use Amazon Titan Embeddings v2 for cost-optimized vectorization, and Nemotron-3 via OpenRouter for reasoning. We store the vectors locally in ChromaDB.

To prevent memory crashes from massive autogenerated files like `package-lock.json`, we built a Fat Repo Shield middleware that hard-caps parsing at 1MB per file, and we use async semaphores to handle rate limits gracefully during ingestion.

JUDGE ASKS: "Can someone inject malicious code through a GitHub issue to hijack your AI?"
Q: What stops a prompt injection attack through repository content?
That attack is real — someone plants "ignore previous instructions" inside a public issue, and when our AI reads it, it executes the command. We neutralize this with XML sandboxing. All untrusted content — code, issues, PR comments — is wrapped in strict XML tags before the AI ever sees it. The AI is explicitly instructed to treat everything inside those tags as inert string data that cannot be executed under any circumstance.
JUDGE ASKS: "Why not just use GitHub Copilot?"
Q: What makes this different from GitHub Copilot or Cursor?
Copilot and Cursor are tactical execution tools. They help you write code once you are already inside the correct file. DevLens is a strategic navigation tool. Copilot writes the `for` loop; DevLens tells you which file out of 3,000 actually needs the `for` loop, and what architecture patterns you must follow. They are highly complementary.
JUDGE ASKS: "How do you avoid LLM Hallucinations?"
Q: AI makes up fake code and fake dependencies. How do you stop that?
By strictly isolating responsibilities. The Dependency Map is never generated by the LLM. It is generated by a deterministic `Tree-sitter` parser that builds an exact Abstract Syntax Tree. The structural data is 100% ground truth. The LLM is strictly confined to semantic search and translating dense text into simple analogies. We do not let the AI guess the architecture.
